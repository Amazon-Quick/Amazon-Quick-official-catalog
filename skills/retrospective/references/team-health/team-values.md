# Team Values Retrospective

<Metadata>
Name: Team Values Retrospective
Category: team-health
Aliases: Values Check, Culture Retro, Principles Review
Duration: 60–90 minutes
Team Size: 4–15
Best For: Examining whether the team is living its stated values; grounding abstract principles in concrete behavior through specific examples
Definition: A retrospective examining whether the team is living its stated values and principles.
</Metadata>

<Gotchas>
1. Use when: Bi-annually or annually as a values health check; When culture drift is suspected (behaviors diverging from stated values); After organizational value statements are created or updated; When new leaders join who may have different values interpretation; When decisions are being made that seem to contradict stated values; After a values violation that needs team-level processing
2. Do NOT use when: Team has no explicit values or principles (need to create them first); When values are imposed rather than co-created (address the imposition first); Weekly cadence (too abstract for frequent use; values manifest over longer periods); When the conversation would become performative rather than genuine
3. Performative values discussion: Going through the motions of values review without genuine willingness to admit gaps or change behavior. Fix: The facilitator must model vulnerability first; share a personal example of falling short.
4. Only management decides what values mean: Leaders interpret values unilaterally; team members can't challenge or contribute to interpretation. Fix: Values interpretation must be co-created. Different perspectives on what a value means in practice are valuable, not threatening.
5. Values without behavior: Values remain abstract ("Be innovative!") without concrete behavioral anchors. Fix: For every value, define 2–3 specific observable behaviors that demonstrate it in daily work.
6. Blame-based gap discussion: Turning values gap analysis into finger-pointing at specific people. Fix: Frame as collective responsibility: "WE fell short" not "YOU fell short." If individual issues exist, address them in private.
7. No examples, only abstractions: Discussing values entirely in the abstract without grounding in specific situations. Fix: Every claim must be backed by an example: "When did we live this?" and "When did we not?"
</Gotchas>

<Instructions>
<Workflow - Team Values Retrospective
  description="Step-by-step facilitation guide for Team Values Retrospective."
>
1. [Setup] (Before session)
   - Gather the team's or organization's stated values/principles
   - Print or display them prominently
   - Optionally send pre-work: "Come with one example of when we lived our values well and one example of when we fell short"
   - Prepare the space for honest, reflective conversation

2. [Open] (10 minutes)
   - Read the team/org values aloud together
   - Frame purpose: "Values are only real if they're lived. Today we're examining whether our stated values match our actual behaviors; celebrating where they do and honestly addressing where they don't."
   - Set tone: "This is about our collective behavior, not about calling out individuals. We're asking 'are WE living this?' not 'is someone else failing?'"
   - Acknowledge: "It's uncomfortable to admit when we don't live our values. That discomfort is exactly what makes this valuable."

3. [Stage: Values Inventory] (5 minutes)
   - Purpose: Reconnect with what the stated values actually are and what they mean
   - Prompt: "Looking at our values - what does each one mean to you? Are we interpreting them the same way?"
   - Facilitation tip: Quick check - do people have the same understanding of each value? "Customer obsession" might mean different things to different people. Surface interpretation differences before evaluating behavior against them.

4. [Stage: Lived Values; Positive Examples] (20 minutes)
   - Purpose: Ground values in concrete positive examples and reinforce values-aligned behavior
   - Prompt: "For each value - when did we live it well recently? Share a specific moment, decision, or action where this value was clearly guiding our behavior."
   - Facilitation tip: Start here ALWAYS. Positive examples create psychological safety for the harder conversation and build a concrete picture of what "living the value" looks like. Capture specifics: who, what, when, and the impact. These become the behavioral reference library.

5. [Stage: Values Gaps; Where We Fell Short] (25 minutes)
   - Purpose: Honestly identify situations where actions didn't align with stated values
   - Prompt: "Where did we fall short of our values? When did we make a decision or take an action that contradicted what we say we stand for? What happened?"
   - Facilitation tip: This requires safety and honesty. Focus on situations and decisions, not individuals. Explore with curiosity: "Why did we deviate? Was it pressure? Unclear application of the value? A genuine conflict between two values?" Most gaps aren't willful violations; they're unconscious drift under pressure.

6. [Stage: Root Causes of Gaps] (15 minutes)
   - Purpose: Understand WHY values gaps occurred so they can be prevented
   - Prompt: "For our biggest gaps - what caused the deviation? External pressure? Competing priorities? Unclear what the value meant in that situation? Were two values in conflict?"
   - Facilitation tip: Common root causes: (a) Time pressure made the value feel like a luxury, (b) The value conflicted with another value in that situation, (c) The value was too abstract to apply concretely, (d) There was no accountability mechanism. Each has a different remedy.

7. [Stage: Recommitment & Behavioral Specifics] (15 minutes)
   - Purpose: Strengthen commitment with specific behavioral definitions
   - Prompt: "How do we better live our values going forward? What specific behaviors would demonstrate each value in our daily work? What would we do differently next time we face a similar situation?"
   - Facilitation tip: Convert abstract values into concrete behaviors. "Customer obsession" → "Before any decision, we ask 'how does this affect the customer?' and someone must represent the customer perspective." Write these behavioral translations alongside the value statements.

8. [Synthesize] (5 minutes)
   - Summarize: values confirmed, positive examples captured, gaps identified, behavioral specifics added
   - Update the values document with behavioral anchors
   - Confirm any structural changes needed (accountability, decision frameworks)

9. [Close] (10 minutes)
   - Read the values with their new behavioral anchors
   - Round: "Which value do you personally commit to living more intentionally?"
   - Confirm next review cadence
   - Appreciation: "Living values honestly, including admitting gaps, IS living our values"
</Workflow - Team Values Retrospective>
</Instructions>

<Resources>
- The "positive examples library" is an underrated output. Concrete stories of values in action are far more powerful than value statements alone for onboarding, culture communication, and daily reinforcement.
- When two values conflict in a situation (e.g., "Move fast" vs. "Quality first"), don't pretend there's no tension. Explore: "When these two values conflict, which takes priority in which situation?" Document the priority framework.
- If the team or organization has values that are clearly not lived at a systemic level (and the team has no power to change it), acknowledge the gap honestly rather than pretending. "This is an organizational aspiration that we individually can't solve" is valid.
- Distinguish between: (a) values we aspire to (growth goals), (b) values we actually live (current culture), and (c) values we claim but violate (hypocrisy risk). All three deserve honest examination.
- Bi-annual or annual cadence is right for this format. Values manifest over months, not weeks. More frequent reviews become performative because not enough behavioral evidence accumulates between sessions.
</Resources>
